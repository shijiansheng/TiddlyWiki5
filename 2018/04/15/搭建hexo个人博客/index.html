    安装Hexo

    • 鼠标右键任意位置，选择 Git bash Here，输入： npm install hexo -g 文件会安装在 C:\Users\用户名\AppData\Roaming\npm\node_modules 文件夹下。
      可以通过npm命令 npm root -g 查看。

    • 安装好了随便找个位置建个名叫 hexo 的文件夹，在这个文件夹里右键选择 Git bash Here，输入 hexo init 和 npm install 命令来初始化文件和安装依赖包。

    • 此时分别输入 hexo g 和 hexo s （生成静态网页和本地预览命令）即可查看初始页面。

    Hexo下mathjax的转义问题

    问题

    • _的转义，在markdown中，_是斜体，但是在latex中，却有下标的意思，就会出现问题。
    • \\的换行，在markdown中，\\会被转义为\，这样也会影响影响Mathjax对公式中的\\进行渲染。

    原因

    hexo默认使用marked.js去解析我们写的markdown，比如一些符号，_代表斜体，会被处理为<em>标签，比如x_i在开始被渲染的时候，处理为x<em>i</em>，这个时候mathjax就无法渲染成下标了。比如粗体*,也是无法在mathjax渲染出来的，好在有替代，\同理。hexo使用的markdown引擎的不是很好，很多其它引擎在这方面都可以处理。

    解决方法

    网上有四种种方法：手动escape、保护代码块、更换Hexo的markdown引擎、修改Hexo渲染源码。
